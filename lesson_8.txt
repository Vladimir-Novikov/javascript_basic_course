Не выполняя код, ответить, что выведет браузер и почему:

if (!( "a" in window )) {
    var a = 1 ;
   }
alert(a);

a in window == true , т.к. это название переменной, которой присваиваем 1 и она есть в коде, т.е. a находится в окне.
Но т.к. есть !, то получаем false, значит в условие не заходим, и переменной а ничего не присваиваем.

Получаем вывод undefined. Переменная а не определена.
--------------------------------------------
var b = function a (x) {
    x && a(--x);
    };
alert(a);

получаем undefined, т.к. браузер за первый проход по коду обнаружил объявление переменной b, а при выполнении второго прохода функция, видимо ни на что не ссылается и получаем ошибку.
если бы мы не передавали ссылку на функцию в переменную b, то получили бы функцию, как в следующем примере.
Но в данном случае alert ничего не вывела, а ошибка была в консоли.
--------------------------------------------
function a (x) {
    return x * 2 ;
    }
var a;
alert(a);

alert выводит функцию. 
var a - мы объявили переменную, но т.к. ничего ей не передали, то в ней так и осталась функция.
----------------------------------------------
function b (x, y, a) {
    arguments [ 2 ] = 10 ;
    alert(a);
    }
b( 1 , 2 , 3 );

вызываем функцию b и передаем три аргумента (1, 2, 3), но в функции присваиваем аргументу с индексом 2, т.е третьему == а, значение 10, с помощью псевдо массива arguments.
Вывод alert 10

function a () {
    alert( this );
    }
a.call( null );

получаем [object Window]. Т.к. передали функции call null в качестве первого параметра, который используется в качестве параметра this.
Получается, мы получили это окно в качестве объекта.
